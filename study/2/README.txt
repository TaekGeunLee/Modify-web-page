sub1.html
: 전체 body에 특정 width 값을 준 다음에 간단한
  레이아웃을 짰다. 이전에 웹디자인기능사 시험 볼 때에도
  이런 식으로 짠 기억이 있다만.. (물론 합격은 했다)
  
  브라우저를 최대화 시킨 상태에서,  
  body의 width보다 현재 쓰고 있는 기기의 화면 크기가 크다면, 
  약간의 여백이 남고, 그 반대라면 아래쪽에 스크롤이 생긴다.
  
  직접 body의 width를 조정하면서 
  변화를 관찰해보자. (body의 색상을 gray로 지정했다.)
  
  모니터 기종마다 화면의 크기는 제각각이기에
  확실히 이상적인 디자인은 절대 아니다.
  
sub2.html
: 이번엔 body에 width를 주지 않고 레이아웃을 짰다.
  
  이렇게 되면 width에 고정값을 주지 않는 것이기 때문에
  다양한 기종의 화면에 대응할 수 있다.
  
  최상단에 수평으로 배치된 div 컴포넌트(.logo, .nav)들의 
  css 스타일들을 살펴보자.(11~12번째 문단)
  
  width에 유동값인 %를 줌으로써 
  브라우저의 크기를 조절할 때마다 그에 맞춰서 크기가 변하는 것을
  관찰할 수 있다.
  
  이전 sub1.html 파일보다 확실히 나아졌음을 알 수 있다.
  
되도록이면 body 같은 전체 레이아웃에 
고정값(px값)을 주지 않도록 하자.
